Title: myFlix

Description:

A movie app that allows users to view information about different movies, genres, and directors.
Users can also create an account, update their information, and add movies to their list of favorites.

Business Logic URL HTTP Method Request Body data format Response Body data format
LIST ALL MOVIES
Responds with a list of ALL movies and their associated data to the user. Each move includes Title, Description, Genre, Genre Description, Director Name, Director Bio, Director Birthdate, Image URL, and whether it is a featured film.		 /movies GET None A JSON object holding data about all movies, structured like:
{
"Title":"Movie Title Here",
"Description":"Movie Description Here",
"Genre": {
"Name":"Genre Name Here",
"Description":"Genre Description Here",
},
"Director": {
"Name":"Director Name Here",
"Bio":"Director Bio Here",
"Birth":"YYYY.MM.DD",
},
"imageURL":"Image URL Here",
"Featured":true/false
}
SINGLE MOVIE INFORMATION
Given a movie title, responds with all data on that movie. Includes Title, Description, Genre, Genre Description, Director Name, Director Bio, Director Birthdate, Image URL, and whether it is a featured film.		 /movies/[title] GET None A JSON object holding all data about the given movie.
GENRE INFORMATION
Given a genre, responds with a description of the genre		 /movies/genre/[genre] GET None A text message describing the given genre
DIRECTOR INFORMATION
Given a director, responds with data about that director (name, bio, birth year)		 /movies/directors/[director] GET None A JSON object holding all the data about the given director. Includes Name, Bio, Birthdate
ADD USER
Adds a new user, passes name and favorite film, responds with name, favorite film, and unique user ID		 /users POST A JSON object holding data about the user to add, structured like: A JSON object holding data about the user that was added, including a unique ID
UPDATE USER NAME
Given a user id and username, updates the username, responds with text message indicating the action was completed or failed		 /users/[id] PUT None A text message indicating that the username was updated
ADD FAVORITE MOVIE
Given a user ID and a movie title, adds the title to the user's favorite movie list, responds with confirmation message that the movie has been added		 /users/[id]]/[title] POST None A text message indicating a movie was added to the user's favorite list
REMOVE FAVORITE MOVIE
Given a user ID and a movie title, removes the title from the user's favorite movie list, responds with confirmation message that the movie was removed.		 /users/[id]]/[title] DELETE None A text message indicating a movie was removed from the user's favorite list
REMOVE USER
Given a user ID, removes the user from the database, responds with a confirmation message that the user was removed		 /users/[id] DELETE None A text message indicating the user email was removed
